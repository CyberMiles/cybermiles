The CyberMiles foundation is entering an Intellectual Property (IP) agreement with the 5miles software development team to Open Source the blockchain software 5miles have developed so far. The source code will be released on Github once the agreement finalizes in early 2018.

The CyberMiles blockchain is a Smart Contract platform specifically optimized for e-commerce applications. By that, we mean that, in collaboration with other Open Source projects, we are solving Ethereum's performance, scalability, and developer productivity problems.

Performance & scalability

The entire Ethereum blockchain network can only process around 20 transactions per second (TPS) today, making it woefully inadequate for e-commerce applications. There are several technically sound and competitive proposals to improve Ethereum performance. However, the Ethereum network is now too big to make dramatic changes quickly. That opens a window of opportunity for new public blockchains.

By using a modern DPoS consensus engine, such as the Tendermint, instead of Ethereum's PoW engine, we can potentially improve blockchain performance to 10,000 TPS. Then, through network architecture design pioneered by Cosmos and Polkadot, we can horizontally partition the network to deploy multiple public blockchains -- each optimized for a specific set of application scenarios while sharing a common base token (i.e., the CMT in our case).

Together with our Open source collaborators, we believe that the performance and scalability problems that have long plagued public blockchains will be solved in the next 6-12 months.

Developer productivity

The Ethereum Smart Contract is a ground breaking technological innovation. However, its implementation is immature. Ethereum users and investors have lost hundreds of millions of dollars assets due to obvious bugs in Smart Contracts. And a recent code review shows that the Ethereum Smart Contracts have a bug occurrence rate far higher than industry average. These systematic problems reflect the poor design and implementation of the virtual machine and supporting libraries around Ethereum Smart Contracts.

From the history of enterprise software, we learnt that an Open source project is only accepted as an enterprise / professional solution when there is a full stack of well supported software tools, libraries, frameworks, and execution environments around it. The full stack establishes coding best practices and norms that dramatically reduce errors and bugs, and improve developer productivity.

At CyberMiles, we will design and implement a full stack of enterprise support software, including a rules engine, a business process manager, a distributed database, a new virtual machine, to run Smart Business Contracts off the blockchain when necessary (and only record the results on the chain).
